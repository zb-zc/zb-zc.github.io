    头文件与类的声明

    类的形式一般分为两种，一种使带指针的类，一种使不带指针的类；

    头文件的布局

    #ifndef __COMPILE__
#define __COMPILE__ 

1. 前置声明  
2. 类声明
3. 类定义

    类的声明

    class complex  //class head
{
public:        //class body
complex (double r = 0, double i = 0): re (r), im (i) { }
complex& operator += (const complex&);
complex& operator -= (const complex&);
complex& operator *= (const complex&);
complex& operator /= (const complex&);
double real () const { return re; }
double imag () const { return im; }
private:
double re, im;

friend complex& __doapl (complex *, const complex&);
friend complex& __doami (complex *, const complex&);
friend complex& __doaml (complex *, const complex&);
};

    模板的引出

    对于复数来说，有整数型的实部虚部，还有浮点型的实部虚部。上面声明的是浮点数的类，所以为了适用所有的数据类型，引出了模板的使用。 

    template<typename T>
class compile
{
    public:
    ...
    private:
    T re,im;
    ...
}

    模板的使用如下：

    {
    complex<double> c1(2.5,1.5);
    complex<int>    c2(2,6);
    ...
}

    inline 内联函数

    函数若在class body内定义完成，便会自动形成inline函数，如果在class外定义就不是inline函数

    访问级别

    public:使外界能够使用的函数以及数据
private:数据部分应该封装起来，不被外界使用调用

    构造函数

    创建一个对象时自动运行的函数

    • 构造函数的目的：创建一个对象

    • 构造函数的名字要跟类的名称相同

    • 可以拥有默认参数

    • 没有返回值类型

    • 构造函数特有的语法
      初始化初值列 冒号：

        complex (double r = 0, double i = 0): re (r), im (i) { }

      相当于： 

       complex (double r = 0, double i = 0) {
       re (r), im (i)
   }

    • 构造函数可以有多个 -重载overloading，但是以下情况是冲突的 

        complex (double r = 0, double i = 0): re (r), im (i) { }
  complex(): re (r), im (i) { }

      因为在下面两种创建对象的情况 

        complex c1;
  complex c2();

    参数传递与返回值

    把构造函数放到private里面

    这样操作的作用：不允许外界创建对象；比如单例模式

    常量成员函数 （const）

    在函数后面加const ,如下所示： 

    double real () const { return re; }
double imag () const { return im; }

    class 类里面分为会改变数据的和不会改变数据的函数，只要是不会改变数据的直接加上const，比如上面的函数，只是为了返回re的值，不会去改变re的值，所以加上了const
    创建一个对象如下

    complex c1(2,1);
cout << c1.real();

    但是如果没有加const，下面这种情况编译就会出现问题 

    const complex c1(2,1);
cout << c1.real();

    创建对象的时候声明 c1的参数是不可以改变的，但是在complex类中如果没有加const就说明 re是可以修改的。所以编译的时候就会报错

    参数传递

    • pass by value:传递所有数据
    • pass by reference(引用):在底层就相当于传递指针 “&”。这种速度比较快
      （尽量传引用）
    • return by value :返回数据
    • return by reference: 在可以的情况下尽量用引用的方式返回
      什么情况下不可以 return by reference
      在函数结束后，变量被释放，这种情况下不可以将此变量的引用传递出去，除了这种情况，其他情况都可以返回引用。

    friend(友元)

    在下面的类中，友元起的作用： 

       class complex  //class head
{
public:        //class body
complex (double r = 0, double i = 0): re (r), im (i) { }
complex& operator += (const complex&);
complex& operator -= (const complex&);
complex& operator *= (const complex&);
complex& operator /= (const complex&);
double real () const { return re; }
double imag () const { return im; }
private:
double re, im;

friend complex& __doapl (complex *, const complex&);
friend complex& __doami (complex *, const complex&);
friend complex& __doaml (complex *, const complex&);
};

    友元可以自由取得friend的private成员

    • 相同class的各个对象互为友元如下：

      class complex
  {
      public:
      complex (double r = 0, double i = 0): re (r), im (i) { }
      int func(const complex& param){
      return param.re;    
      }

  private:
  double re,im;    
  }
  complex c1(2,1);
  complex c2;
  c2.func(c1);

    操作符重载

    在c++中操作符就是一种函数。

    成员函数 this

    所有的成员函数都带有this指针，指向成员函数属于的对象

    操作符重载（类成员函数）

    两个操作数的 例如“+=”

    • 规则： 左边的是调用者，右边是被调用者，如下

        inline complex&
      __doapl (complex* ths, const complex& r)
      {
      ths->re += r.re;
      ths->im += r.im;
      return *ths;
      }
  inline complex&
      complex::operator += (const complex& r)
      {
      return __doapl (this, r);
      }
  
  complex c1;
  complex c2;
  c2 += c1 //c2 = c2 + c1;

    如上：this 指的是c2（左值），r指的是c1

    • 传送者无需知道接收者是以reference形式接受，比如

      c2 += c1
      在下面重载函数中 c1对应的重载函数中的形参，但是形参的类型是reference，c1是一个value。这就是所说的传送者无需接收者是以reference形式接受。
      还有一种情况是函数返回值无需知道函数返回类型是reference。如下 

        inline complex&
          __doapl (complex* ths, const complex& r)
          {
          ths->re += r.re;
          ths->im += r.im;
          return *ths;
          }

    操作符重载 （非成员函数）

    inline complex
operator + (const complex& x, const complex& y)
{
return complex (real (x) + real (y), imag (x) + imag (y));
}

inline complex
operator + (const complex& x, double y)
{ 
return complex (real (x) + y, imag (x));
}

inline complex
operator + (double x, const complex& y)
{
return complex (x + real (y), imag (y));
}
    • 以上的函数不可return by reference; 返回一个对象，对象在函数执行完毕之后分配的临时空间被释放。
    • typename + ():创建临时对象。
